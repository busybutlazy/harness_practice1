name: scaffold-fullstack
description: 當你需要建置或修改這個最小化的 React + FastAPI Todo 學習專案時使用。僅限此示範用 repo 觸發，不可用於無關的 repo。
---

目標：
建立或更新一個以前後端分離為基礎、組成最少、容易理解的 Todo 示範專案，並以 Docker 作為執行與驗證方式。

流程：
1. 先檢查目前 repo 的結構，確認前端、後端與 Docker 相關檔案的現況。
2. 判斷本次任務影響前端、後端，或兩者皆有。
3. 若任務同時涉及前端與後端，要求 Codex 啟動：
   - 一個前端子代理
   - 一個後端子代理
   - 一個審查子代理
4. 只實作完成需求所必需的最小變更，避免擴張範圍。
5. 小心整合各子代理的結果，避免不必要的重寫。
6. 以 Docker 執行並驗證最終應用程式可正常運作。
7. 回傳最終結果時，提供變更檔案、執行命令與已知限制。

實作準則：
- 後端 API 必須保持簡單，並使用記憶體內資料，不引入資料庫。
- 前端 UI 必須保持精簡、可操作、可完成 Todo 基本流程。
- 前後端應維持清楚分離。
- Docker 設定應以最小可行方案為原則，避免加入不必要的服務或複雜設定。
- 除非任務明確要求，否則不要加入額外基礎設施或產品化設計。

交付內容：
- 已變更的檔案
- Docker 執行命令
- 已知限制

限制：
- 不使用資料庫
- 不加入驗證機制
- 必須使用 Docker
- 不使用 CSS 框架
- 除非原本就已存在，否則不要使用 TypeScript
